# Frontline Rally Command

Godot 4.5 stable .NET + C#/GDScript로 만든 540×960 세로형 아이소메트릭 군단 전선 시뮬레이션입니다. 시뮬레이션 코어는 C#, HUD·입력·FX는 GDScript이며 보병·공성병기·드래곤·건물은 CC0 3D 모델을 자체 베이크한 스프라이트입니다. Android 릴리스 메타데이터는 **1.5.1 (version code 16)** 입니다.

## 실험 브랜치: 30대30 개별 판단 AI

`codex/mini-battle-agent-ai`는 출시 게임과 분리한 30대30 방패병 병목 실험입니다. 이 브랜치에서는 `scenes/agent_battle_lab.tscn`이 기본 장면이며, 유닛마다 Node를 만들지 않고 하나의 C# 시뮬레이션이 60개 고정 배열 에이전트를 처리합니다. `AGENT AI / BASELINE / PAUSE / 1X·2X / RESET` 버튼으로 개별 판단과 단순 중앙 전진을 즉시 비교할 수 있습니다.

```bash

PATH=/private/tmp/godottest1-dotnet9:$PATH \
DOTNET_ROOT=/private/tmp/godottest1-dotnet9 \
  /private/tmp/godot45mono/app/Godot_mono.app/Contents/MacOS/Godot \
  --headless --path . -s tests/run_agent_battle_lab.gd

PATH=/private/tmp/godottest1-dotnet9:$PATH \
```

무인 캡처는 `build/smoke_agent_battle_lab.png`에 540×960으로 저장됩니다. 고정 시드 결과, 수용 기준 판정, 성능 수치와 한계는 [30대30 개별 판단 AI 실험 보고서](docs/agent-battle-lab-results.md)에 기록했습니다. 이 브랜치는 실험용이며 프로덕션 병합이나 모바일 실기기 성능을 의미하지 않습니다.

## 한 판 규칙과 조작

- 44×88(3,872타일) 전장을 위쪽 빨강과 아래쪽 파랑이 나눠 가지며, 420초 안에 적 HQ를 파괴하거나 육지 영토 92%를 확보하면 즉시 승리합니다. 시간 종료 시에는 점유율, HQ 체력 비율, 잔존 병력 체력 순으로 판정합니다.
- 중앙에는 고정 시드로 만든 하나의 큰 대칭 호수가 있습니다. 물은 중립이며 점유율에서 제외되고 지상군·건물·지상 플로우를 막지만, 드래곤은 호수와 절벽을 무시하고 직행합니다. 양쪽 육지 회랑은 항상 HQ까지 연결되는지 생성 시 검증합니다.
- 유닛과 건물은 자기 HQ까지 이어지는 열의 보급선을 점유합니다. 한번 전진해 얻은 영토는 부대가 옆 열로 이동해도 유지되고, 상대 진군만 같은 보급선을 다시 덮어 탈환합니다. HUD 점유율과 타일 색은 이 하나의 영속 소유권 배열을 그대로 읽습니다.
- 전 셀은 고정 시드 `PackedByteArray` 고도 0/1/2를 가지며 언덕·정상·절벽이 상하 180° 대칭으로 생성됩니다. 고도차 1은 통행 가능하고 오르막 속도는 0.7배, 고도차 2는 통행 불가 절벽입니다. 별도 장애물 타일은 없으며 생성 직후 모든 배치 후보에서 상대 HQ까지의 경로를 검증합니다.
- 지상군은 상대 HQ에서 역산한 팀별 가중 플로우 필드를 따라 전역 우회합니다. 거리·병력 혼잡·오르막 비용을 합산하고 절벽 전이는 제외하므로 완만한 길을 선호합니다. 건물 건설·파괴 즉시, 그리고 0.75초마다 밀도를 갱신해 한 길이 붐비면 다른 경사로를 선택합니다. 대각선 코너 끼어들기를 막는 공통 전이 판정, 반경별 절벽 여유폭, 다음 셀 중심 포털 조향, 0.4초 실제 진행 정체 시 9×9 결정론적 복구 경로를 함께 사용해 돌출 절벽 모서리에서도 옆 길을 찾아 빠져나옵니다. 의도적인 WAIT와 제자리 공격은 막힘으로 오인하지 않습니다.
- 전장 끝에서도 플로우 목표가 상대 HQ이므로 유닛은 자동으로 본진을 찾아가며, 비행 드래곤은 고도·절벽과 건물 충돌을 무시하고 HQ로 직행합니다.
- 고지에서 저지를 공격하면 피해 1.25배, 저지에서 고지는 0.75배, 같은 높이는 1.0배입니다. 원거리 보병은 자기 고도가 1 이상이면 사거리 +0.5셀을 받습니다. 건물은 고도와 무관하게 자기 영토 빈 타일에 건설할 수 있습니다.
- 양측 HQ는 방어탑과 같은 3.6셀 사거리·0.8초 간격으로 자동 방어하며, 한 발 피해는 방어탑 6의 정확히 3배인 18입니다. 근접 보병은 비행 드래곤을 탐지·유지·공격할 수 없고, 원거리병·방어탑·HQ는 대공 공격이 가능합니다.
- 시작 골드는 파랑 180/빨강 120, 자동 수입은 초당 2.25, 처치 보상은 6입니다. 팀별 생존 유닛 상한은 300명이며 30명 단위마다 자동 수입과 처치 보상이 10%p씩 감소해 300명에서는 0%가 됩니다. HUD의 `AI L1~L5` 버튼은 적 수입을 1.0/1.25/1.5/1.75/2.0배로 바꾸며 재시작 기본값은 L3=1.5배입니다.

게임을 열면 먼저 `MELEE / RANGED / SIEGE / DRAGON` 설정 화면이 표시됩니다. HP·피해·공격주기·사거리·탐지·속도·반경·생산주기·생산량·건설비·병과별 상성 배율과 방패/SIEGE 전용 값을 세부 조정할 수 있으며 한 프로필이 양 팀에 똑같이 적용됩니다. 큰 반경 유닛은 스포너 외곽 링의 충돌 없는 위치에서 생산되고, SIEGE 탐지 범위를 공격 사거리보다 늘리면 먼 적을 향해 접근하되 사거리 진입 전에는 발사하지 않습니다. `DEFAULTS`는 출시 기본값, `COPY SETTINGS`는 정렬된 `schema_version: 1` JSON을 기기 클립보드에 복사하고, 유효한 `START` 전에는 시간·건설·맵 입력이 모두 정지합니다. 재시작도 같은 설정 화면으로 돌아옵니다.

파랑 육지 영토의 빈 타일을 짧게 탭/클릭해 `MELEE 60`, `RANGED 80`, `SIEGE 140`, `DRAGON 220` 스포너, `RALLY 80`, `TOWER 120` 중 하나를 건설합니다. 설정 화면에서 바꾼 4병과 건설비는 HUD 버튼과 안내문에 즉시 반영됩니다. 건설 시간은 가격×0.1초(기본 6/8/14/22/8/12초)이며 20% HP에서 시작해 완성될 때까지 생산·공격·집결 기능이 정지합니다. 건설 중에도 공격받아 파괴될 수 있습니다. 완성된 스포너는 지정 병과를 계속 생산하고, 새 유닛은 가장 가까운 집결지를 찾아갑니다. 방어탑은 파랑 HQ 중심 5×5 안에서만 설치됩니다. 모바일은 핀치/드래그, 데스크톱은 휠/왼쪽 드래그를 사용하며 확대 범위는 1.0×~16.0×입니다.

건물 최대 HP는 일반 스포너 1200, 집결지 1300, 방어탑 1600, 드래곤 레어 1500, HQ 12000입니다. 건설 프리뷰는 지면에 붙은 골조와 진행률로 표시되며 완성 순간 팀색 펄스가 발생합니다.

## 집결지, 군단과 진형

- 전진 집결지는 대기 병력 20명이 모이면 현재 병과 구성 그대로 군단을 편성해 상대 HQ로 출격합니다. 4.2셀 집결 반경과 집결 전용 이동은 선두 병력이 통로를 막아 숫자가 7–8에서 멈추던 병목을 제거합니다.
- 방어 집결지는 최대 28명을 주변 진형에 주둔시키고 적이 접근하면 교전한 뒤 재정렬합니다. 28명 초과분은 별도 전진 군단으로 즉시 출격하며, 집결 반경을 벗어날 때까지 고정 주둔대에 대한 WAIT를 우회하고 분리력을 낮춰 실제 병력이 통과합니다.
- `LINE`은 근접병을 넓은 전열, 원거리·SIEGE를 후열에 놓고, `WEDGE`는 근접 선두를 좁은 쐐기로 만들며, `LOOSE`는 전 병과를 두 배 간격으로 퍼뜨려 같은 SIEGE 폭발에 겹치는 인원을 줄입니다.
- 앵커는 기존 플로우 필드와 선택 웨이포인트를 따라가며, 군단 속도는 편성 병과 중 최저 속도입니다. 이동 앵커와 깃발은 실제 생존 대원 중심보다 1.75셀 이상 앞설 수 없어 병력이 막혔을 때 깃발만 사라지는 현상이 없고, 유닛은 회전된 슬롯을 추종하면서 기존 분리·혼잡·WAIT·관성을 사용합니다.
- 적을 감지하면 `ENGAGED`로 전환해 개별 seek/공격을 허용하고 슬롯 힘을 약화합니다. 교전 종료 1.2초 뒤 재정렬하며, 생존자가 출격 인원의 30% 미만이면 `BROKEN`이 되어 기존 무소속 행동으로 돌아갑니다.
- 집결지의 숫자는 현재 대기/주둔 인원입니다. 전진은 깃발+화살, 방어는 방패 링으로 구분됩니다. 군단 배너는 사각형 디버그 마커가 아니라 폴·꼭지·바람에 휘는 노치형 천을 하나의 절차형 메시로 그리며 팀과 상태를 반투명 색으로 표시합니다. 집결지가 파괴되면 주둔·대기 병력은 `BROKEN` 폴백으로 개별 행동합니다.

## 병종과 데이터 구조

| 항목 | 근접 | 원거리 | SIEGE | 드래곤(비행) |
|---|---:|---:|---:|---:|
| HP | 48 | 20.4 | 40 | 520 |
| 속도 | 1.015 | 0.875 | 0.56 | 1.19 |
| 스포너 생산 주기 | 5.76초 | 5.76초 | 12.343초 | 45초×2기 |
| 사거리 | 0.72 | 2.2, 고지 2.7 | 1.2~14.0 | 0.9 |
| 반경 | 0.14 | 0.13 | 0.26 | 0.38 |
| 피해 / 간격 | 10 / 0.65초 | 8 / 0.80초 | 55.8 AoE / 3.2초 | 36 / 0.90초 |
| 공격 대상 | 지상만 | 지상+공중 | 지상+공중+건물 | 지상+공중 |

SIEGE는 1.2~14.0셀의 밀집 지점을 골라 0.9초 곡사탄을 쏘며, 예고 링 안 1.8셀을 피해 55.8로 중심 100%→가장자리 40% 공격합니다. 일반 사거리와 AoE는 타겟 반경을 포함하고 아군 오사는 없습니다. 분리는 두 유닛 반경 합×1.2를 목표 간격으로 써 대형 병기가 보병 위에 겹치지 않습니다.

근접병은 2.5셀 안에서 적 원거리병을 만나면 청록 방패 테두리가 켜지고, 적 원거리병이 모두 3.0셀 밖으로 나갈 때 해제됩니다. 방패 중 이동속도는 20%로 낮아지지만 원거리 공격으로 받는 피해는 10%만 적용되며 근접·SIEGE·드래곤 피해는 줄이지 않습니다. 원거리병은 표적 둘레의 16개 사격 후보를 예약하고 지형·혼잡·아군 예약을 고려해 측면으로 전개합니다. 후열에 실제 옆 사격 자리가 있으면 정지한 전열을 80%, 자신을 20% 비율로 비켜 움직여 여러 명이 화망을 만들고, 한 칸 복도처럼 옆자리가 없을 때만 겹치거나 진동하지 않고 대기합니다.

상성 배율은 지형·폭발 감쇠와 곱연산됩니다: RANGED→MELEE 1.7, MELEE→SIEGE 1.6, MELEE→RANGED 1.2, SIEGE→MELEE 1.5, RANGED→SIEGE 0.7, RANGED→DRAGON 1.2, DRAGON→RANGED 1.7, DRAGON→SIEGE 1.5, MELEE→DRAGON 0.6, 나머지는 1.0입니다. 유리 상성 타격은 더 큰 주황 강타 스파크로 표시됩니다.

`BattleSimulation.cs`는 30 Hz 고정 틱과 논리 `(col,row)` 좌표만 사용합니다. 유닛과 군단은 각각 C# 고정 용량 SoA 배열과 ID 풀에 저장되며 유닛·군단 Node, 시그널, `_physics_process`가 없습니다. `GroundNavigation.cs`가 공통 지형 전이·반경 여유폭을, `BattleSimulation.FiringPositions.cs`가 고정 배열 사격 예약과 비대칭 양보를 담당합니다. `BattleSimulation.Legions.cs`는 최근접 집결지 배정, 회전 슬롯, 앵커와 상태 전이를 맡고, `BattleSimulation.Ai.cs`는 양 팀에 재사용 가능한 관측형 AI 상태기계를 제공합니다. 기존 공간 버킷·플로우·전투 코드는 그대로 재사용합니다. SIEGE 투사체도 풀링된 지연 데이터이며 폭발 후보는 상대 팀 공간 버킷만 검사합니다.

렌더러는 전역 Y 정렬 지상군 MultiMesh, 팀별 드래곤 MultiMesh, 공유 그림자, 군단 배너와 집결 고스트 배치를 일괄 처리합니다. C#이 여섯 개의 인터리브 `PackedFloat32Array`를 MultiMesh 포맷으로 직접 조립하고 GDScript는 각 버퍼를 한 번 대입할 뿐 유닛·군단별 getter를 호출하지 않습니다.

유닛 HP바는 풀피에서 숨고 피해를 입은 순간 3초 표시된 뒤 마지막 0.6초 동안 페이드합니다. 폭과 그림자, 렌더 스케일은 병과별 반경 하나에서 파생되며 발끝 앵커가 논리 좌표와 셀 고도에 접지됩니다. 건물과 HQ의 HP바는 항상 표시됩니다. 영토·체커·그리드는 액터보다 채도와 명도를 낮추고, 높은 타일은 밝은 상판과 어두운 절벽 측면으로 구분합니다. 탭 픽킹은 실제 화면에 보이는 상승 다이아몬드 폴리곤을 검사합니다.

## 집결지 밸런스, AI와 성능

Godot 4.5 자동 밸런스 경로는 기본 AI L3에서 파랑이 아무것도 짓지 않을 때 **323.1초 DEFEAT**(파랑 점유율 22.2%), 시작 후 AI L1을 선택하고 파랑 자동 지휘를 켠 적극 대응 경기에서 **420.0초 VICTORY**(파랑 점유율 52.0%)를 기록했습니다. 두 경로 모두 목표 5–7분 안에 결론이 나며 HQ HP는 양측 12000입니다.

이전 AI의 실패 원인은 `CountBarracks >= 3`에서 영구 반환해 첫 3개 건물 뒤 경제·카운터 판단을 모두 멈춘 것이었습니다. 개편 직후에는 집결지가 가장자리 열에 지어지고 1.7셀 반경에 대기열 선두 3–4명만 들어가 출격 10명을 영원히 못 채우는 두 번째 결함도 계측으로 확인했습니다. 실기기 재현에서는 40명이 집결지에 배정돼도 선두를 본 39명이 행군용 WAIT에 걸려 카운트가 7–8에서 멈추는 추가 병목을 확인했으며, 집결 이동만 WAIT를 우회하도록 분리했습니다. 방어 정원 28명 뒤의 초과 출격군도 같은 WAIT에 영구 고착되는 동안 독립 앵커만 이동하던 원인을 재현해, 출구 우선권과 1.75셀 앵커 결속으로 수정했습니다.

동골드 카운터 매트릭스 20회 결과는 근접전 MELEE→SIEGE **20/20**, 개활지 SIEGE→MELEE **20/20**, DRAGON→RANGED **20/20**, DRAGON→SIEGE **20/20**입니다. 방패 상태 RANGED→MELEE의 실효 배율은 `1.7×0.1=0.17`입니다. AI 건전성 50+50 결과는 무행동 플레이어 **50/50 패배, 평균 403.6초**, AI 대 AI는 원시 파랑 승리 **37/50**이지만 **50/50 모두 타임업**이므로 무승부 보정 점수율 **50%**, 평균 **420.0초**, 골드 적체/건설 정지 **0건**입니다.

20개 군단×12명(240유닛) 전용 스트레스는 틱 평균 **0.603 ms**, p95 **2.523 ms**, C# 렌더 스냅샷 평균 **0.074 ms**를 기록했습니다. 군단 ID→인덱스 조회는 고정 테이블 O(1)이며 실제 대원 중심도 틱당 유닛 한 번의 O(n) 스캔으로 캐시합니다. 일반 600/1500/3000기 스트레스와 보드 30셀 델타 계측도 함께 유지합니다.

- 영토는 0.2초마다, 건물 이벤트 때는 즉시 공간 버킷으로 갱신하고 점유율도 같은 시점에 캐시합니다.
- 유닛은 3개 그룹을 라운드로빈으로 나눠 표적·분리·혼잡 판단만 스태거링하고, 이동·쿨다운·타격·타겟 생존 검사는 매 틱 유지합니다.
- 고빈도 hit/shot/death는 타입별 PackedArray 소유권 이전 채널을 쓰며, 사소 FX는 프레임당 40개로 제한하고 SIEGE/HQ/승패 이벤트는 제한하지 않습니다. HP바 타이머는 피해 이벤트 때만 갱신합니다.
- 공개 언어 경계는 프레임당 `Step`, `GetRenderSnapshot`, `DrainEvents`, `GetHudSnapshot`, 정수형 `GetBoardVersion`과 필요 시 `TryBuild`/`GetBoardDelta` 벌크 호출만 사용합니다. 전체 `GetBoardSnapshot`은 경기 초기 1회만 호출하며, 테스트 전용 명령·스냅샷도 유닛별 반복 호출 없이 묶어서 전달합니다.
- 프로젝트의 데스크톱·모바일 렌더러는 `gl_compatibility`로 고정돼 있습니다.

## 보드 렌더링 성능 패스 3

현재 맵은 **44×88, 3,872타일**입니다. 타일 상판은 하나의 3,872-instance `MultiMeshInstance2D`가 처리하며 transform은 초기 한 번만 기록합니다. 중앙 호수도 같은 인스턴스의 water custom data와 셰이더 잔물결로 표시되고, 정적 해안선·절벽 측면은 CanvasItem이 한 번 그린 뒤 다시 만들지 않습니다. 전선은 캐시된 전용 레이어만 소유권 버전 변경 때 갱신합니다. 소유권 변경은 C#의 변경 인덱스/새 소유자 PackedArray 델타로 전달되어 해당 instance color/custom data만 씁니다. 0.62초 영토 플래시는 타일 custom data의 시작 시간과 셰이더 `TIME`으로 감쇠하므로 셀별 FX 객체와 매 프레임 CPU draw가 없습니다.

2026-07-21 Apple M4, Godot 4.5 stable .NET, OpenGL Compatibility에서 같은 30셀 동시 플립을 3회 워밍업 후 18회 측정했습니다. 변경 전은 968타일 전체 `_draw` 재테셀레이션과 30개 `territory_change` FX, 변경 후는 3,872타일 장면에서 버전 조회→30셀 델타→MultiMesh instance 갱신과 강제 렌더를 포함합니다. 확대된 보드의 고정 프레임 기준도 별도로 측정해 실제 소유권 변경 증분을 분리합니다.

| 30셀 동시 플립 | 변경 전 전체 draw+FX | 변경 후 delta MultiMesh |
|---|---:|---:|
| 전체 평균 | 10.843 ms | **0.846 ms** |
| 전체 p95 | 11.182 ms | **1.045 ms** |
| 보드 경계 평균 | 0.047 ms | **0.103 ms** |
| 렌더 갱신 평균 | 10.795 ms | **0.793 ms** |
| 렌더 갱신 p95 | 11.134 ms | **0.988 ms** |
| 고정 보드 렌더 p95 | 미측정 | **0.959 ms** |
| 소유권 플립 증분 p95 | 미측정 | **0.029 ms** |

전체 렌더 p95는 타일 수가 4배가 된 상태에서도 이전 전체 재테셀레이션보다 **91.1% 감소**했습니다. 소유권 변경이 더하는 p95 0.029 ms가 스파이크 목표 ≤2 ms를 달성하며, 절대 0.988 ms 중 0.959 ms는 변경이 없는 44×88 보드의 고정 렌더입니다. GitHub 공유 러너의 CPU software-renderer 편차를 고려해 CI 증분 게이트는 60 ms의 회귀 방지 한도로 분리하지만, 구조 테스트가 정확히 30개 instance만 갱신되고 전체 sync·transform 재기록·영토 FX 객체가 없음을 별도로 강제합니다.

군단 시각 자가검증:

1. 집결: 집결지 숫자와 진형 슬롯으로 대기 병력이 모이는 과정이 보입니다.
2. LINE: 근접 전열이 가장 넓고 원거리·SIEGE가 진행방향 뒤에 섭니다.
3. ENGAGED: 양측 배너 사이에서 슬롯 힘이 약해지고 기존 seek/런지/타격 FX가 보입니다.
4. LOOSE: 같은 12명이 LINE보다 두 배 간격으로 퍼져 SIEGE 링 중첩 인원이 줄어듭니다.
5. WEDGE: 근접 선두가 삼각형으로 좁아지고 지원 병과와 드래곤이 후미를 따릅니다.
6. BROKEN: 30% 미만 생존자는 배너를 잃고 개별 플로우 이동으로 전환됩니다.
7. 집결지 모드: `smoke_rally_advance.png`는 절차형 깃발과 20명 출격, `smoke_rally_defend.png`는 방패 링과 28명 주둔, `smoke_rally_defend_egress.png`는 고정 방어대 사이를 실제 초과 병력과 깃발이 함께 빠져나온 상태를 보여줍니다.
8. 호수·건설·FX: `smoke_lake_overview.png`는 중앙 단일 호수와 육지 회랑, `smoke_construction.png`는 접지된 골조와 진행률, `smoke_tactical_fx.png`는 반투명 트레이서·링·파편을 보여줍니다.
9. 설정 화면: `smoke_match_settings.png`는 HUD가 비치지 않는 불투명 세로 화면, 네 병과 탭, 스크롤 필드와 고정 하단 `DEFAULTS / COPY SETTINGS / START`를 보여줍니다.
10. 이동·화망: `smoke_cliff_detour.png`는 절벽 돌출부 양옆으로 갈라진 지상군을, `smoke_firing_line.png`는 측면 전개한 여러 궁병·방패 테두리·동시 사격 FX를 보여줍니다.
