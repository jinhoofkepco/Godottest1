# Mini Battle AI Lab: 30대30 다중 케이스 결과

## 결론

`codex/mini-battle-agent-ai`는 프로덕션 게임과 분리된 30대30 방패병
개별 판단 실험 앱이다. 동일한 60개 고정 배열 에이전트를 네 지형에서
Baseline과 Agent로 비교했고, 세 고정 시드 및 반복 결정성 검사를 모두
통과했다.

- 실행 수: 24개 기본 조합 + 12개 Agent 반복 = 36회
- 시드: `230723`, `230724`, `230725`
- 케이스: `BOTTLENECK`, `CORNER_TRAP`, `ROUTE_CHOICE`, `OPEN_CONTROL`
- 결과: `AGENT SCENARIO MATRIX PASS`
- 로컬 C# 빌드: 경고 0, 오류 0
- 기존 룰 회귀: `RULE TESTS PASS`

CORNER_TRAP의 Agent 진입은 시드별 19/12/14명이며 팀별 최소 6명을
충족했다. 원시 12초 탈출률은 8~26%, 최대 체류는 17.3~87.0초다.
다만 이 수치에는 협로 안에서 의도적으로 교전하거나 사망한 병력이
포함된다. 실제 비의도 대기는 1.5~15.0 agent-seconds, 최대 무진전은
2.5~4.5초로 코너 진동/고착 기준을 통과했다. OPEN_CONTROL은 강제
대칭화 없이 생존자 차이 2/1/1명, 잔여 HP 차이 32/16/32를 기록했다.

전투 HP, 피해, 공격 간격, 사거리, 스폰 위치와 양 팀 인원은 비교 내내
변경하지 않았다. 독립 리뷰 뒤 `OPEN_CONTROL`의 쌍별 명령·위치·피해
동기화와 `CORNER_TRAP`의 교전·후퇴 금지를 모두 삭제했다. 네 지형은
동일한 결정·이동·전투 코드를 사용한다. 표적 재예약은 양 팀의 같은
로컬 순서를 사용하고 매 결정 주기마다 선처리 팀만 교대한다.

## 한계

로컬 수치는 Apple M4의 Godot 4.5 .NET 실행 결과다. Android APK 내부
서명·패키지·arm64 C# 어셈블리는 CI 아티팩트에서 검사하지만, 이번
실험에서 실기기 프레임 시간·발열·배터리·GC는 측정하지 않았다. 이
결과는 60명 미니 실험의 구조 비교이며 프로덕션 600명 성능 수치로
해석하면 안 된다.
